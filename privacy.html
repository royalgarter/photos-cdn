Privacy Policy

Last updated: July 5, 2026

Summary: Photos CDN processes text queries and image data to deliver images. We do not sell personal data. Query strings sent to the API may be stored in server logs for operational purposes.

1. Who We Are

Photos CDN ("we", "our", "the Service") is an AI-powered image delivery platform. This Privacy Policy explains what data we collect when you use our API or web interface, how we use it, and your rights regarding that data.

2. What Data We Collect

When you use the Photos CDN API or web playground, we may collect the following:

  • Query text — the text= parameter you send with API requests (e.g., "sunset over mountains")
  • Request metadata — IP address, HTTP method, URL path, response code, timestamp, and user-agent string
  • Generated images — AI-generated images are stored in our ArangoDB database and Cloudflare R2 object storage alongside their embedding vectors and source metadata
  • Settings — if you configure API keys via the Settings panel (localhost only), those credentials are stored in ArangoDB encrypted at rest

We do not collect names, email addresses, payment information, or any personally identifiable information unless you explicitly provide it.

3. How We Use Your Data

  • To fulfill API requests — embedding your query, searching the database, generating images
  • To improve search relevance — stored images and their embeddings improve the quality of future vector searches
  • For operational monitoring — server logs help diagnose errors and monitor system health
  • To comply with legal obligations

We do not use query text to train AI models directly. Query text is passed to third-party AI providers (Gemini, Cloudflare Workers AI, Pollinations.ai, HuggingFace) to generate image embeddings or images; those providers have their own privacy policies.

4. Third-Party Services

The Service integrates with the following third-party providers. Each may process query text or images according to their own terms:

  • Google Gemini — embeddings and image generation (Google Privacy Policy)
  • Cloudflare Workers AI & R2 — image generation and object storage (Cloudflare Privacy Policy)
  • Pollinations.ai — free-tier image generation
  • HuggingFace — fallback image generation (HuggingFace Privacy Policy)
  • Pexels, Unsplash, Pixabay — stock photo indexing via their public APIs

5. Data Retention

  • Server logs — retained for up to 30 days for operational purposes, then purged
  • Generated images — stored indefinitely unless you request deletion or the database is reset
  • API keys stored in Settings — retained until you update or delete them via the Settings panel

6. Cookies and Tracking

The Photos CDN web interface does not use tracking cookies, analytics scripts, or third-party advertising. The only client-side scripts are Alpine.js (UI reactivity) and Lucide Icons (SVG icons), neither of which sends data to external servers.

7. Data Security

We use industry-standard practices to protect data in transit (HTTPS/TLS) and at rest (ArangoDB and Cloudflare R2 encryption). No method of transmission or storage is 100% secure; we cannot guarantee absolute security.

8. Your Rights

Depending on your jurisdiction, you may have the right to:

  • Request deletion of images or logs generated from your API calls
  • Request a copy of data associated with your use of the service
  • Object to processing of your data

To exercise these rights, contact us using the information below.

9. Children's Privacy

The Service is not directed at children under 13. We do not knowingly collect personal information from children. If you believe a child has provided us with personal data, please contact us and we will delete it promptly.

10. Changes to This Policy

We may update this Privacy Policy from time to time. The "Last updated" date at the top of this page reflects the most recent revision. Continued use of the Service after changes constitutes acceptance of the updated policy.

11. Contact

Questions about this Privacy Policy? Reach us at the project repository or via the contact details in the service metadata.
